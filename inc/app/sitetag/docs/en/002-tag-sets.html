Tag Sets

A tag set is a collection of tags used for a defined goal. By default, SiteTag comes with a tag set names keywords that allow you to add a generic tagging system to sitelllite pages. Each time you need a distinct list of tags, you should create a new tag set.

The tag set definition file

Tag sets are defined into ini files located into /inc/app/sitetag/conf/sets/. For example, the default keywords tag set is named keywords.ini.php. Here is the content of this file:

description = "A global keywords list for sitellite_page"
sitellite_owner = admin
sitellite_team = none
kind = global
view = public
tag = private
edit = private

template = off
The following table describe each parameters in details:

Parameter
 Default value
 Description
description
This field describe what is this tag set. I will be used in a future admin interface for tag sets.
sitellite_owner
 admin
 Who is the owner of this tag set. The owner has a full read and write access to the tag set.
sitellite_team
 none
 The team that owns this tag set. Only used in complex situations.
kind
 global
 global: every users share the same keywords.
members: this tag set only is used by logged users. Each user has its own set of tags to manage.
view
 public
 The access level to view the tags in this tag set.
tag
 private
 The access level to add of remove tags to an item. A user with only this access level cannot create new tags. Read access is sufficient to be able to tag.
edit
 private
 The access level to add, and edit items to tag, and to add or remove tags. Read access is sufficient to be able to edit tags.
template
 off
 Template to use to display the list of items related to a specific tag.

This way, you can create many different tag sets, with different levels of permissions. On the same Sitellite installation, you could have a tag set every user can modify, another reserved to logged users, and another user only by site administrators.
